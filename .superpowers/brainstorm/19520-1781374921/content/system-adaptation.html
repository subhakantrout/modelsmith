Safety Features

🛑 Pre-flight check — Estimates RAM/disk before any operation

💾 Checkpointing — Every node saves progress, resume after crash
📊 Live monitor — RAM/CPU/disk usage bar during operations
🔄 Graceful fallback — If RAM hits 90%, auto-switch to disk streaming
🧪 Dry run mode — Simulates pipeline, shows peak memory estimate
✅ Core Philosophy
The tool should NEVER crash the user's system. Every operation knows its memory budget,
every pipeline has a fallback, and every user gets a pipeline tuned to THEIR hardware.

This is the systems-mindset foundation. Everything else (UI, backend, node types) builds on top of this. Click or type — does this approach feel right?
