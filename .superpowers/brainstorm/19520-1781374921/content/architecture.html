Model Capability Editor — Concept Overview

A visual tool to edit any local model's capabilities: uncensor, enhance, compress

High-Level Architecture

1. LOAD MODEL
Pick any local model (GGUF, SafeTensors, HF format)
↓
2. ANALYZE & VISUALIZE
See refusal directions, layer sensitivity, capability map, size breakdown
↓
3. EDIT CAPABILITIES (Visual Canvas)
🧹 Abliterate
Remove censorship
🧬 Merge
Combine with other models
📦 Inject LoRA
Add specific skills
⚡ Compress
Quantize / prune for RAM
↓
4. PREVIEW & TEST
Side-by-side comparison: original vs edited, built-in chat
↓
5. EXPORT
Save as GGUF, SafeTensors, LoRA adapter, or HuggingFace push

Key Design Decisions

Backend Python + PyTorch (wraps Heretic/Abliterix, mergekit, llama.cpp)
Frontend Web UI (Gradio? React? Tauri?)
Target Local/consumer hardware — CPU-friendly with optional GPU
Format support GGUF, SafeTensors, HF Transformers, LoRA adapters

First question — what frontend approach feels right?

Since this is a visual editing tool, the frontend choice matters

A

Gradio Web UI

Python-native, fastest to build, runs locally in browser. Good for prototyping. Limited visual sophistication.

B

Tauri + React Desktop App

Native feel, modern UI libraries (node-based editors, d3), distributable as .exe/.dmg. More work upfront.

C

Electron + React

Same as Tauri but heavier. Easier Python-backend communication via child process. Familiar ecosystem.
